## Observacoes

- Protecao no `middleware.ts`: politica hibrida com `admin/*` (exceto `/admin/login`) redirecionando para `/admin/login?from=...` e rotas de usuario (`/perfil`, `/minhas-reservas`, `/dashboard*`, `/dashboard-rsv`, `/pricing/dashboard`) redirecionando para `/login?redirect=...`.
- Token admin validado por assinatura/expiracao (`admin_token` JWT), sem comparacao literal fixa.
- Classificacao de placeholder: apenas `/dashboard/routes` (texto explicito de funcionalidade futura).
- Em `/admin/cms` e `/flash-deals` existem frases "em breve", mas em contexto de estado/partes secundarias; por isso permanecem como **Implementada**.
- Plano de acao V2 para rotas ausentes: `docs/MAPEAMENTO_ROTAS_RSV360_V2_PLANO_ACAO.md`.
- `/api/docs` e uma rota tecnica de API (nao `page.tsx`), validada no smoke com resposta HTTP 200.
